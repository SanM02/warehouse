ORDEN DE EJECUCION
================================================================================

1. Editar archivo .env (agregar dominio a ALLOWED_HOSTS)
2. Editar nginx.conf (agregar dominio a server_name)
3. Editar index.html (titulo + tag favicon)
4. Editar app.component.html (nombre en navbar)
5. Reemplazar favicon.png (si el cliente lo proporciona)
6. Reconstruir frontend:
   
   docker-compose up -d --build frontend

7. El usuario debe editar hosts de Windows (requiere admin)
8. Probar acceso en: http://ferreteria-jg.local


================================================================================

INSTRUCCIONES PARA EL CLIENTE (ARCHIVO HOSTS)
================================================================================

CONFIGURAR DOMINIO PERSONALIZADO EN WINDOWS:

1. Abrir Bloc de Notas como ADMINISTRADOR
   (Click derecho en Bloc de Notas -> Ejecutar como administrador)

2. Ir a Archivo > Abrir

3. En la barra de direccion escribir:
   C:\Windows\System32\drivers\etc

4. Cambiar "Documentos de texto" a "Todos los archivos"

5. Seleccionar el archivo "hosts" (sin extension)

6. Agregar al final del archivo:
   127.0.0.1    ferreteria-jg.local

7. Guardar (Ctrl + S) y cerrar

8. Abrir navegador y acceder a:
   http://ferreteria-jg.local

NOTA: Esto solo funciona en ESTA computadora.
Para otras PCs de la red, repetir el proceso en cada una.


================================================================================

NOTAS IMPORTANTES
================================================================================

1. FAVICON: El cliente debe proporcionar su logo. Si no tiene, puede crear
   uno gratis en: https://favicon.io

2. DOMINIO .LOCAL: Solo funciona en la PC donde se configure el archivo hosts.
   Es una solucion local, no un dominio de internet.

3. REBUILD OBLIGATORIO: Despues de los cambios, ejecutar:
   docker-compose up -d --build frontend

4. BACKEND NO SE TOCA: Solo necesita el ALLOWED_HOSTS actualizado via .env,
   y se aplicara al reiniciar el contenedor.

5. VERIFICAR: Despues de aplicar cambios, probar:
   - http://localhost (debe seguir funcionando)
   - http://ferreteria-jg.local (nuevo acceso)
   - http://localhost:4200 (alternativo)
